# ECDSA Key Finder Scripts

Коллекция Python скриптов для поиска и анализа ECDSA ключей в больших CSV файлах.

## 📁 Описание файлов

### 1. `ecdsa_key_finder.py`
**Базовый скрипт** - Полная загрузка данных в память для небольших файлов.

### 2. `ecdsa_key_finder_optimized.py`
**Оптимизированный скрипт** - Потоковое чтение файла без загрузки в память.

### 3. `ecdsa_fast_finder.py` ⭐ **РЕКОМЕНДУЕТСЯ**
**Сверхбыстрый скрипт** - Использует grep для молниеносного поиска в больших файлах.

## 🚀 Логика работы

1. **Пользователь вводит значение k** (например, 14)
2. **Система ищет строку с указанным k**, определяет значение r
3. **Находит строку где**: k=введенное_k, r=найденное_r, s=r, z=r
4. **Выводит результат** в табличном формате с дополнительной колонкой "Формула"

## 📊 Пример вывода

```
k	r	s	z	d	ks+1	ks+z	(ks+1)/k	(ks+z)/k	d-k+1	(d-k+1)/k	((d-k+1)/k)^77	d-k+z	(d-k+z)/k	((d-k+z)/k)^77	Формула
14	5	5	5	13	71	75	22	11	0	0	0	4	68	43	x
```

## ⚡ Быстрый запуск

### Рекомендуемый способ (самый быстрый):
```bash
python3 ecdsa_fast_finder.py
```

### Альтернативные способы:
```bash
# Оптимизированная версия
python3 ecdsa_key_finder_optimized.py

# Базовая версия (для небольших файлов)
python3 ecdsa_key_finder.py
```

## 📋 Требования

- **grep** (для быстрого скрипта, обычно предустановлен в Linux/macOS)
- **CSV файл** `escda79.csv` в том же каталоге

## 💡 Использование

1. Запустите скрипт

3. Получите результат в табличном формате
4. Для выхода введите `quit` или нажмите `Ctrl+C`

## 🛠️ Устранение неполадок

### Ошибка "grep не найден":
```bash
# Ubuntu/Debian
sudo apt install grep

# CentOS/RHEL
sudo yum install grep
```

### Файл не найден:
Убедитесь что `escda79.csv` находится в том же каталоге что и скрипт.

### Проблемы с кодировкой:
Скрипты автоматически используют UTF-8. Если проблемы остаются, проверьте кодировку файла.

## 🔍 Детали реализации

### Алгоритм поиска:
1. **Первый поиск**: `grep "^k;"` - находит любую строку начинающуюся с k
2. **Извлечение r**: Парсинг второй колонки найденной строки
3. **Второй поиск**: `grep "^k;r;r;r;"` - находит точную строку с условиями
4. **Форматирование**: Преобразование и вывод результата

### Безопасность:
- Таймауты для предотвращения зависания
- Проверка входных данных
- Обработка исключений
- Безопасный выход по Ctrl+C
