# Исследование ECDSA: Поиск формул для нахождения k по r

## Обзор проблемы

**Кривая:** y² = x³ + 7 mod 79  
**Базовая точка:** Q = (1, 18)  
**Порядок группы:** n = 67  

Цель: найти методы определения значения k (секретного nonce) по известному значению r из ECDSA подписи.

## Математическая основа

В ECDSA:
- r = (kQ).x mod n, где kQ - результат умножения базовой точки Q на скаляр k
- s = k⁻¹(z + rd) mod n, где z - хеш сообщения, d - приватный ключ
- Подпись: (r, s)

**Проблема:** По известным r, s, z найти k.

## Найденные методы решения

### 1. Метод прямого поиска (Brute Force)

**Принцип:** Перебор всех возможных значений k от 1 до n-1.

```python
def find_k_brute_force(r):
    for k in range(1, 67):
        point = k * Q
        if point.x % 67 == r:
            return k
```

**Преимущества:**
- Гарантированно находит все решения
- Простота реализации

**Недостатки:**
- Сложность O(n)
- Неэффективен для больших n

### 3. Baby-step Giant-step алгоритм

**Принцип:** Метод meet-in-the-middle для дискретного логарифма.

**Алгоритм:**
1. Выбрать m = ⌈√n⌉
2. Baby steps: вычислить jQ для j = 0,1,...,m-1
3. Giant steps: для i = 0,1,...,m-1 проверить, есть ли целевая точка среди baby steps

**Сложность:** O(√n) времени и памяти

### 4. Анализ квадратичных вычетов

**Принцип:** Использование свойств кривой для определения валидных координат.

Для точки (r, y) на кривой:
- y² ≡ r³ + 7 (mod 79)
- Проверяем, является ли r³ + 7 квадратичным вычетом

**Формула проверки:** Символ Лежандра (a/p) = a^((p-1)/2) mod p

### 5. Обнаруженные математические закономерности

#### 5.1 Комплементарные пары
**Наблюдение:** Для многих r значения k часто идут парами (k, 67-k).

**Примеры:**
- r=1: k=[1, 66], [19, 48] 
- r=15: k=[3, 64]
- r=42: k=[5, 62]

**Формула:** Если k является решением, то часто 67-k тоже решение.

#### 5.2 Линейные соотношения
AI-анализ выявил некоторые линейные связи:
- r ≡ 5k + 9 (mod 67) для отдельных случаев
- r ≡ 3k + 2 (mod 67) для некоторых пар

#### 5.3 Статистические свойства
- Корреляция r-k: -0.0361 (слабая отрицательная)
- Распределение k на r: 2-4 значения k на каждое r
- 31 уникальное значение r из возможных 66

## Практические формулы

### Основная формула поиска k
```python
def find_k_from_r(r):
    """Главная функция поиска k по r"""
    # Метод 1: Прямой поиск в таблице
    if r in r_k_lookup:
        return r_k_lookup[r]
    
    # Метод 2: Математический подход
    y_squared = (r**3 + 7) % 79
    if is_quadratic_residue(y_squared, 79):
        # Найти y координаты и дискретный логарифм
        return find_discrete_log(r, y)
    
    # Метод 3: Брутфорс как последний шанс
    return brute_force_search(r)
```

## Рекомендуемый алгоритм

### Гибридный подход:
1. **Первый шаг:** Проверка в предвычисленной таблице (O(1))
2. **Второй шаг:** Анализ комплементарных пар
3. **Третий шаг:** Baby-step Giant-step для неизвестных r
4. **Последний шаг:** Полный перебор

```python
class EfficientKFinder:
    def find_k(self, r):
        # Шаг 1: Lookup
        if r in self.lookup_table:
            return self.lookup_table[r]
        
        # Шаг 2: Комплементарные пары
        complement_candidates = self.find_complement_pairs(r)
        if complement_candidates:
            return complement_candidates
        
        # Шаг 3: BSGS
        return self.baby_step_giant_step(r)
```

## Выводы и рекомендации

1. **Для практического применения:** Используйте предвычисленную таблицу lookup - самый быстрый и точный метод.

2. **Для больших кривых:** Комбинируйте Baby-step Giant-step с анализом паттернов.

3. **Для исследований:** Изучайте комплементарные пары и линейные соотношения.

4. **Безопасность:** Данные методы работают только для малых кривых (n < 100). Для реальных криптографических кривых эти атаки неприменимы.
